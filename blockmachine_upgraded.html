Architecture

How blockmachine works

From request routing and market allocation to verification, scoring, and emissions.

01

Miners

Suppliers of infrastructure. Each miner runs full or archive nodes and competes to serve live RPC traffic. Miners bid a USD-denominated price per compute unit and are evaluated continuously on latency, uptime, reliability, and correctness.

The market mechanismThere is no allocation, no queue, no scheduling. Miners set a price, the gateway measures their performance, and the market decides who gets work.
02

Gateway

The single entry point for all requests. It authenticates, meters, routes, and logs. The gateway decides which miner gets each request based on quality and price.

Routing scoreA miner that fails the quality threshold gets almost nothing. The routing score combines price and quality into a single allocation — visible in real time.
03

Clients

The client receives verified responses through a standard JSON-RPC endpoint. No tokens, no Bittensor, no miner selection. Just an API that works.

04

Epoch boundary

An epoch is 360 blocks — roughly 72 minutes. Everything that happened is frozen, finalised, and handed to validators for scoring. No real-time judgement. No live intervention.

05

Verification

Validators re-execute sampled requests against trusted reference infrastructure. The miner's logged response is compared against ground truth.

Deterministic verificationIf the reference node returns a different hash, the miner lied. Match or mismatch — no threshold, no benefit of the doubt.
06

Correctness enforcement

If a miner's response doesn't match the reference — even once — the miner is permanently banned. Not penalised. Not warned.

Banned, not warnedA single incorrect response ends a miner's participation. This creates the strongest possible correctness incentive.
