01 / Platform
Platform overview
3 questions

Blockmachine is a blockchain RPC (Remote Procedure Call) infrastructure provider powered by Bittensor. It provides fast, independently verified access to full and archive blockchain nodes via a standard JSON-RPC interface — no custom SDK, no proprietary API, no lock-in.

Unlike centralized RPC providers (Alchemy, QuickNode, dRPC), Blockmachine routes requests through a decentralized network of competing miners. Every response is cryptographically verified against trusted reference nodes, so correctness is a guarantee — not a trust assumption.

Key differentiatorCompeting miners drive prices down. Deterministic verification means you don't have to trust the provider — you can verify it yourself.

Blockmachine currently supports Bittensor (TAO) as its primary chain, with expanded multi-chain support on the roadmap.

⬤ Bittensor (TAO) — Live → Ethereum — Phase 5 → BSC — Phase 5 · Additional EVM chains — Planned

All supported chains are accessed via standard JSON-RPC 2.0 — the same interface you use today with any other provider.

02 / Trust
Verification & correctness
3 questions

Blockmachine uses deterministic hash verification. At the end of every epoch (~72 minutes), validators sample logged miner responses and re-execute the same requests against trusted reference infrastructure.

The miner's response hash is compared to the reference node's response hash. Because blockchain state is deterministic — the same block height, same method, same parameters always produce the same result — any mismatch is proof of a lie.

ConsequenceA single mismatched response permanently bans the miner. Not a warning. Not a score penalty. Banned. This creates the strongest possible correctness incentive.

An epoch is 360 Bittensor blocks — approximately 72 minutes at 12 seconds per block. It is the fundamental unit of the verification and scoring cycle.

During an epoch, the gateway routes live traffic, logs all miner responses, and accumulates performance metrics. At the epoch boundary, everything freezes: logs are archived (public and immutable), validators sample and verify responses, miners are scored, and weights are updated on-chain. Then the next epoch begins.

You don't have to take our word for it. Every epoch's sampled logs are public and immutable — you can audit the verification process yourself. The performance dashboard at blockmachine.io/performance shows live and historical correctness metrics per node.

The economic design also enforces this: miners earn TAO emissions proportional to routing share. A banned miner earns nothing. The cost of lying exceeds any potential benefit, making correctness the rational choice for every miner.

Pricing & plans

Blockmachine charges per Request Unit (RU). Four plans are available:

$0/moFree · 10K RU · 1 key
$25/moPro · 20M RU · 3 keys
$125/moBusiness · 100M RU · 5 keys
$999/moEnterprise · 1B RU · 10 keys

All plans include archive node access, usage dashboard, USDC payments, and the Trace API. Overage rates decrease with tier: $3/1M (Pro) → $2/1M (Business) → $1/1M (Enterprise).

A Request Unit (RU) measures the computational work required to serve an RPC request, normalized across methods and chains. This ensures you pay for the actual complexity of a call — not a flat fee per request.

As a rough guide: simple reads like eth_blockNumber or eth_getBalance cost ~1 RU. Standard queries cost ~5–10 RU. Archive lookups and trace calls (debug_traceTransaction, trace_*) cost ~20–50 RU.

This is a structural feature of the market mechanism — more competition means lower prices for customers. Announced pricing changes will always be communicated before they take effect.

04 / Dev
Integration & API
4 questions

Point your application at one of these endpoints and authenticate with your API key:

Endpoints HTTP: https://rpc.blockmachine.io
WebSocket: wss://rpc.blockmachine.io
Auth header: Authorization: Bearer $BLOCKMACHINE_API_KEY

It is standard JSON-RPC 2.0 — no SDK, no custom client, no special configuration. Store your API key server-side only and never expose it in browser code.

Blockmachine uses three non-standard JSON-RPC error codes in addition to the standard -32600 to -32700 range:

  • -32029
    Rate limit exceeded
    Retry with exponential backoff. Not terminal.
  • -32021
    Invalid API key
    Terminal — surface to a human. Check key rotation.
  • -32030
    RU budget exhausted
    Terminal — surface to a human. Upgrade plan or wait for renewal.

Three habits cover the majority of key security risk:

1. Server-side only. Never include your API key in browser JavaScript, mobile app bundles, or any client-side code. Use environment variables (process.env.BLOCKMACHINE_API_KEY) and read them at runtime on the server.

2. One key per environment. Use separate keys for development, staging, and production. If a key is compromised, rotation affects only one environment.

3. Budget-scoped keys for AI agents. If an autonomous agent will call the API, provision a dedicated key with a low RU budget from the dashboard. The gateway hard-stops when the budget is hit — no runaway spend.

06 / Network
Bittensor & mining
3 questions

Blockmachine is built on a Bittensor subnet. Miners in the subnet run full or archive blockchain nodes and compete for routing share. Validators score miners each epoch based on latency, uptime, correctness, and price, then set weights on-chain — determining each miner's share of TAO emissions.

This creates a self-reinforcing incentive loop: miners earn more TAO by delivering fast, correct, competitively priced responses. Bad actors (slow, incorrect, or overpriced) receive less routing, earn less TAO, and face permanent bans for incorrect data.

As a customerYou see none of this. No tokens, no wallet, no Bittensor complexity. Just a standard JSON-RPC endpoint that works — backed by economic incentives you can audit.

The gateway calculates a routing score for each miner that combines two dimensions: quality (latency, uptime, correctness) and price (USD per compute unit). Miners with better quality and lower prices receive proportionally more traffic.

There is no manual allocation, no queue, and no committee. The market decides. A miner that fails the quality threshold gets almost no traffic. A miner that bids too high gets undercut by competitors. The routing score is visible in real time on the Performance dashboard.

Miner onboarding is currently in Phase 3 of Blockmachine's roadmap — external miner registration is opening now, alongside public miner performance dashboards and full documentation.

To mine on Blockmachine, you run full or archive blockchain nodes, register with the Bittensor subnet, and set a USD-denominated price per compute unit. Your earnings are proportional to your routing share, which is determined by your performance and pricing relative to other miners.

Full miner documentation, tooling, and registration are available through the Miners link in the footer, or at team@blockmachine.io.
